# P11: 投资交易复盘 (Autopsy)

## 适用场景

定期复盘，区分"运气"与"实力"。解决：「亏钱了不知道为什么，赚钱了以为是自己牛逼」的错觉。

## 提示词模板

```
# Role
你是一名 **冷酷的交易审计员**。
请对我刚刚结束的一笔交易 **[{{ticker}}]** 进行"尸检"。目标不是安慰我，而是找出系统漏洞。

# Context (User Input)
1. **Ticker:** {{ticker}}
2. **Result:** {{result}}
3. **Original Thesis:** {{original_thesis}}
4. **Exit Reason:** {{exit_reason}}

# Audit Protocol

## 1. 知行合一检查
* 买入理由和卖出理由是否匹配？(例如：基于基本面买入，却因为图形坏了恐慌卖出 = 违规)。

## 2. 运气 vs 实力
* 赚钱是因为逻辑兑现了，还是因为大盘都在涨(Beta红利)？
* 亏钱是因为逻辑证伪了，还是因为止损设置过紧被洗出去了？

## 3. SOP 优化
* 这笔交易暴露了 SOP 流程中的哪个缺陷？(如：调研不深、FOMO入场、不止损)。

# Output Format
* **交易评分:** [0-10分] (基于执行纪律，而非结果盈亏)
* **核心教训:** [一句话总结]
* **改进动作:** "下次遇到类似情况，我应该..."
```

## 输入参数

| 参数 | 说明 | 必填 |
|-----|------|-----|
| `ticker` | 股票代码 | 是 |
| `result` | 交易结果（盈利XX%/亏损XX%） | 是 |
| `original_thesis` | 当初买入的理由（SOP1/SOP2） | 是 |
| `exit_reason` | 实际卖出的理由 | 是 |

## 复盘评分标准

| 评分维度 | 权重 | 说明 |
|---------|-----|-----|
| 知行合一 | 40% | 买卖理由是否匹配 |
| 纪律执行 | 30% | 是否按计划止损/止盈 |
| 逻辑验证 | 30% | 投资逻辑是否得到验证 |

## 常见系统漏洞

| 漏洞类型 | 表现 | 改进方向 |
|---------|-----|---------|
| 调研不深 | 买入后才发现关键风险 | 强化 P1 调研流程 |
| FOMO 入场 | 追高买入 | 强化 P5 审计 |
| 不止损 | SOP 2 变 SOP 1 | 强化 P12 纪律检查 |
| 过早止盈 | PVE 主升浪中被洗出 | 强化 P6 持仓评估 |
