# P4: 个股关注信息清单 (Radar)

## 适用场景

针对关注的个股生成强制关注的信息清单，作为日常搜集信息的"雷达设置"，防止遗漏关键的竞争对手或产业链信号。

## 提示词模板

```
# Role
你是一位拥有20年经验的科技行业首席分析师，擅长产业链穿透分析。

# Task
请针对我关注的标的 **[{{ticker}}]**，生成一份**"强制关注信息清单"**。
这份清单将作为我日常搜集信息的"雷达设置"，防止我遗漏关键的竞争对手或产业链信号。

# Analysis Framework (必须覆盖以下维度)

## 1. 本体核心 (The Core)
- 必须关注的官方文件（如：10-K/Q, 8-K, 月度交付数据）。
- 必须警惕的管理层动作（如：高管增/减持计划、股权激励行权）。

## 2. 直接竞品映射 (The Mirror)
- 列出 3-5 个最直接的竞争对手。
- **关键逻辑：** 当竞品发布财报或新产品时，必须反向思考这对 [Target Ticker] 的市场份额、定价权是利多还是利空？（例如：小鹏降价 -> 蔚来毛利承压）。

## 3. 产业链上下游 (The Chain)
- 上游（成本端）：关注哪些原材料价格？（如：碳酸锂、英伟达芯片供应）。
- 下游（渠道端）：关注哪些分销或政策变化？

## 4. 宏观与行业 (The Macro)
- 利率敏感度、地缘政治风险、特定行业补贴/关税政策。

# Output
请生成一个结构化的 Checklist，列出具体的"关注事件"和"解读视角"。

# Context (User Input)
1. **此次分析的股票标的ticker** {{ticker}}
2. **个股分析详细数据或者附件Knowledge Base Summary:** {{context}}
```

## 输入参数

| 参数 | 说明 | 必填 |
|-----|------|-----|
| `ticker` | 股票代码 | 是 |
| `context` | 已有的个股基石档案或补充资料 | 否 |
