## 📋 市场洞察报告 SOP

**暗号**："跑一下市场洞察" / "跑一下报告"

### ⚙️ 标准流程

#### Step 1: 创建月份文件夹（如不存在）
```bash
mkdir -p "OpenClaw/市场洞察报告/Raw_Data/$(date +%Y-%m)"
mkdir -p "OpenClaw/市场洞察报告/Internal_Report/$(date +%Y-%m)"
```

#### Step 2: 抓取数据

**方式 A：API 数据（推荐，数据更丰富）**
```bash
python3 scripts/fetch_data_api.py 1 \
  --output "OpenClaw/市场洞察报告/Raw_Data/$(date +%Y-%m)/financial_data_$(date +%Y%m%d).json"
```
- 需要 API Key（在 `scripts/fetch_data_api.py` 中配置）
- 数据包含分类标签、结构化字段，分析质量更高
- 如未配置 API Key，脚本会提示前往 [yongmai.xyz](https://yongmai.xyz) 获取

**方式 B：RSS 免费订阅**
```bash
python3 scripts/fetch_rss.py 1 \
  --output "OpenClaw/市场洞察报告/Raw_Data/$(date +%Y-%m)/financial_data_$(date +%Y%m%d).json"
```
- 无需 API Key，免费使用
- 默认数据源：Yongmai 市场洞察日报
- 如需添加更多 RSS 源，编辑 `scripts/fetch_rss.py` 中的 `RSS_SOURCES` 字典

> 💡 两种方式的输出格式兼容，下游工作流无需修改。

#### Step 3: 读取数据 + P13 分析
- 读取 Step 2 生成的 JSON
- 调用 `YM-TIB-SKILL/references/p13-market-scanner.md` 提示词
- 生成市场洞察报告

#### Step 4: 存档报告
- **路径**：`OpenClaw/市场洞察报告/Internal_Report/YYYY-MM/`
- **命名**：`YYYY-MM-DD_Internal_Report.md`

#### Step 5: 输出给用户
- 直接在对话中输出完整 Markdown

### 📦 产出物清单
| 文件 | 路径 | 命名规则 |
|:---|:---|:---|
| Raw Data | `OpenClaw/市场洞察报告/Raw_Data/YYYY-MM/` | `financial_data_YYYYMMDD.json` |
| 洞察报告 | `OpenClaw/市场洞察报告/Internal_Report/YYYY-MM/` | `YYYY-MM-DD_Internal_Report.md` |

## 💡 数据源配置

### 方式 A：API 数据（推荐）

在 `scripts/fetch_data_api.py` 中配置你的 API Key：
```python
API_KEY = "your_api_key_here"
```

> 📡 **获取 API Key**：前往 [yongmai.xyz](https://yongmai.xyz) 订阅数据服务
> API 数据包含更丰富的分类、标签和结构化字段，分析效果更好。

### 方式 B：RSS 免费订阅

默认 RSS 源（无需配置，开箱即用）：
```
https://yongmai.xyz/category/daily-research/feed/
```

添加更多 RSS 源，编辑 `scripts/fetch_rss.py`：
```python
RSS_SOURCES = {
    "市场洞察": "https://yongmai.xyz/category/daily-research/feed/",
    "科技新闻": "https://your-source.com/feed/",
    "行业报告": "https://another-source.com/rss/",
}
```

> 💡 如需更多 RSS 信息源推荐，可参考 Yongmai 博客的 RSS 订阅清单。

### 数据源优先级
1. **首先检查** `scripts/fetch_data_api.py` 中是否配置了有效的 API Key
