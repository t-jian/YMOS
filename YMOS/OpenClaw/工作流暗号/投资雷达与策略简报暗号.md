## 📋 完整分析流程（连续跟踪逻辑）

### 核心定位

投资雷达不是孤立的每日快照，而是**连续跟踪系统**：

> **投资雷达 = 上份雷达的延续 + 最新市场洞察的增量 → 信号演变追踪 + 待办进度更新 + 新行动建议**

### Phase 1: 双源输入

**源1：最新市场洞察（增量信息）**
- `OpenClaw/市场洞察报告/Internal_Report/YYYY-MM/YYYY-MM-DD_Internal_Report.md`
- 提供当日最新市场信号

**源2：上份投资雷达（连续性基础）** ⭐ 关键输入
- `OpenClaw/ 我的持仓与关注点和投资偏好/ai建议/` 最新的投资雷达报告
- 提取：
  - 上期关注板块的状态和监控点
  - 待调研事项列表及进度
  - 行动建议及执行情况
  - 持仓监控价位和预测

### Phase 2: 连续跟踪分析

**1. 信号演变追踪**
对比本期 vs 上期，分析：
- **持续强化**：哪些信号在加强？
- **信号弱化**：哪些逻辑在证伪或冷却？
- **新信号出现**：本期新增的关注点
- **信号消失**：上期关注但本期未验证的

**2. 待办事项进度检查**
- 上期建议的调研是否完成？
- 上期建议的行动是否执行？
- 未完成事项的紧迫性评估

**3. 持仓监控追踪**
- 是否触发监控价位？
- 上期预测是否验证？
- 止盈/止损条件是否达成？

**4. 策略一致性检查**
结合投资档案：
- `我的投资状态卡.md` - 风险偏好、策略定位
- `我的关联偏好.md` - 关注板块、情报关联规则
- `我的持仓.md` - 持仓明细

### Phase 3: 输出保存

**输出路径**：`OpenClaw/ 我的持仓与关注点和投资偏好/ai建议/`

**文件命名：**
- 有重要演变/新信号：`投资雷达_YYYY-MM-DD.md`
- 无重大变化：`运行日志_YYYY-MM-DD.md`

## 💡 与传统分析的区别

| 维度 | 传统单日分析 | 连续跟踪雷达 |
|:---|:---|:---|
| **输入** | 单日市场洞察 | 单日洞察 + 上份雷达 |
| **视角** | 孤立快照 | 趋势演变 |
| **冗余度** | 高（每天重复背景） | 低（聚焦变化） |
| **执行跟踪** | 无 | 有待办进度检查 |
| **预测验证** | 无 | 有上期预测验证 |
